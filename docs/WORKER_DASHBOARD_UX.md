# ROME Worker Dashboard â€” UI/UX Specification

## Overview

The Worker Dashboard is a **personal app** that workers access from their own devices (phones/tablets) to:
1. View their punch history and hours worked
2. Submit time off requests
3. Track the status of their requests

This is separate from the **Kiosk Mode** (shared warehouse computers for clocking in/out).

## Screen 1: Personal Login (`/login`)

### Purpose
Secure login for workers accessing from personal devices.

### Flow
1. Worker enters their **6-digit PIN** (same as kiosk)
2. System sends a **verification code** to their registered email
3. Worker enters the code
4. They're logged in and stay logged in on that device

### Key Elements

1. **Week Summary Card**
   - Total hours this week
   - Progress bar toward 40 hours
   - Daily breakdown

2. **Today Card**
   - Current status (clocked in/out)
   - Clock in/out times
   - Running timer if currently working

3. **Recent Punches**
   - Last 5-7 days of punch history
   - Quick link to full history

4. **Bottom Navigation**
   - Home (current)
   - Time Off (requests)
   - Settings (profile, logout)
